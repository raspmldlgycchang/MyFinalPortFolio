MFC CSocket을 이용한 다중채팅방

문제상황 해결과 구현 방법

출처 : 김관영(본인작성), 작업인원 :1(본인)

개인 간 채팅방 구현에서 문제상황과 해결 방법

한 명의 클라이언트와 한 명의 서버의 대화로부터
리스너 소켓이 다른 클라이언트의 접속을 받기 위해서
현재 연결된 클라이언트와의 대화를 위해서 사용자 소켓이 필요했다.
여러 클라이언트와 한 명의 서버의 대화에서
서버의 사용자 클래스 객체 한 개를 이미 한 명의 클라이언트와의 대화에 사용하고 있는데
다른 클라이언트가 accept된 후 사용하려고 하니 서버에서 이미 사용자 클래스 객체가 사용중이라는 에러가 났다.

해결 : 그래서 사용자 클래스 객체를 연결리스트로 구현하여
클라이언트가 접속할 때마다 동적으로 생성하는 코드를 구현하여
사용자 클래스 객체를 여러 클라이언트가 사용하여 생기는 에러를 해결했다

다중채팅방 구현에서 문제상황과 해결 방법

중간에 클라이언트가 접속을 끊는 경우에 해당 소켓을 제거해야 하는 상황이였다.

해결 : CSocket을 상속받는 사용자 클래스는
클라이언트가 연결을 종료할 때 호출되는 OnClose함수가 있었다.
사용자 클래스 연결리스트를 관리하는 서버 클래스 객체에서
클라이언트가 접속을 끊을 때, 해당 원소를
연결리스트에서 찾아서 이 원소의 OnClose함수를 호출하여 구현했다.
이때 사용자 클래스 연결리스트 노드 전체를 제거하는 함수를 응용하여
this포인터를 이용해서 연결을 끊은 클라이언트를 찾는 방법을 이용했다.
